EVOLUTION-TURNEAMENT

[GENERIC-FUNCTION]
GA
Function Reference
ARGUMENTS:
  - chromosomes initial population [default = ()]
  - num-of-gen number of generations [default = 100]
  - fitness-func fitness function [default = ()]
  - mutation-func mutation function [default = ()]
  - n-best-elements number of best elements [default = 1]
  - threshold threshold [default = 0.75]
  - result best of all/last generation [default = last-generation]

Description:

In the tournament selection, two individuals are randomly chosen from the population.
Therefore, given a random number r between 0 and 1, and a <threshold> value k defined by the user,
if r is less then k the most suitable individual between the two extracted individuals is chosen,
otherwise the worst is chosen.
Subsequently the two individuals are reinserted into the population and a new selection takes place.
In <n-best-elements> it can be defined how many best elements the object returns at the end of the evolution process
If <result> is set to 'all-generations' the object returns <n-best-elements> from each generation,
and only from the last otherwise.

Nell'evoluzione a torneo si scelgono a caso due individui fra quelli presenti nella generazione corrente.
Viene poi generato un numero casuale (r) fra 0 e 1,
se r risulta maggiore di un numero di soglia <threshold> definito dall'utente, viene scelto il peggiore fra i due individui,
altrimenti viene scelto il migliore.
Si procede allo stesso modo fino a chè il numero di individui scelti riempie una nuova generazione
